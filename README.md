# MCLPaper

2D Monte Carlo Localization (MCL) simulation in Python/NumPy with two run modes:

- `visualize`: single live run with matplotlib visualization
- `benchmark`: multi-trial runs with metrics logged to disk

## Instructions (API-Style Reference)

### Requirements

- Python 3.10+
- `uv` package manager

### Install / Run

Dependencies are managed with `uv` via `pyproject.toml` and `uv.lock`.

Run visualize mode (default):

```bash
uv run python main.py --mode visualize
```

Run visualize mode without opening a window (still saves plots):

```bash
uv run python main.py --mode visualize --no-window
```

Run benchmark mode (no plotting, logs metrics to disk):

```bash
uv run python main.py --mode benchmark --trials 50 --steps 300 --particles 2000
```

### CLI Reference (`main.py`)

#### Core

- `--mode {visualize,benchmark}`
- `--steps INT`
- `--particles INT`
- `--seed INT`
- `--dt FLOAT`
- `--motion-noise-v FLOAT`
- `--motion-noise-w FLOAT`
- `--sensor-noise FLOAT`
- `--ray-beams INT`
- `--ray-fov-deg FLOAT`
- `--ray-max-range FLOAT`

#### Simulated Dynamics (Random Acceleration + Jerk)

Control inputs are generated by a stochastic dynamics process:

- maintain `v` and `w`
- acceleration (`a_v`, `a_w`) follows a bounded random walk
- jerk noise drives acceleration updates
- integrate acceleration to update velocity
- integrate `v`, `w` into pose

Flags:

- `--v0 FLOAT`
- `--w0 FLOAT`
- `--jerk-std-v FLOAT`
- `--jerk-std-w FLOAT`
- `--jerk-dist {uniform,gaussian}` (default: `uniform`)
- `--accel-max-v FLOAT`
- `--accel-max-w FLOAT`
- `--v-min FLOAT`
- `--v-max FLOAT`
- `--w-min FLOAT`
- `--w-max FLOAT`

#### Visualize Mode

- Runs a single trial.
- Live matplotlib window shows:
  - particle cloud
  - true robot pose + heading arrow
  - estimated pose + heading arrow
  - ray beams (wall-distance rays, lidar-like)
  - square room walls
  - live RMSE plot
- Visualization updates every `N` steps (`--viz-stride`) to reduce redraw overhead.

Flags:

- `--viz-stride INT` (default: 5)
- `--viz-pause FLOAT`
- `--no-window`
- `--viz-dir PATH`
- `--no-viz-snapshot`
- `--viz-particle-size FLOAT`
- `--viz-particle-alpha FLOAT`

Outputs (default `plots_py/`):

- `mcl_trajectory.png`
- `mcl_rmse.png`
- `mcl_control_dynamics.png`
- `mcl_visualize_live_snapshot.png`

#### Benchmark Mode

- Disables plotting.
- Runs `--trials` independent trials (seeds increment from base seed).
- Saves metrics to timestamped directory.

Flags:

- `--trials INT`
- `--benchmark-dir PATH`

Outputs (default `benchmark_runs/benchmark_YYYYMMDD_HHMMSS/`):

- `metrics.csv`
- `metrics.npz`
- `config.txt`

### Module / Function Reference

#### `main.py`

- `Params`
  - Central configuration dataclass for simulation, visualization, benchmark, and dynamics settings.
- `run_mcl(params, seed=None, step_callback=None)`
  - Runs one MCL trial and returns trajectories, RMSE history, final particles, and control/accel/jerk histories.
- `run_visualize_mode(params, console)`
  - Runs one trial with live matplotlib updates.
- `run_benchmark_mode(params, console)`
  - Runs many trials and writes metrics to disk.

#### `dynamics.py`

- `ControlDynamicsState`
  - Holds simulated control state: `v`, `w`, `a_v`, `a_w`.
- `step_control_dynamics(...)`
  - Updates jerk, acceleration, and velocity with bounds and returns the next control input `[v, w]`.

#### `motion_model.py`

- `motion_model(particles, control, dt, motion_noise_std, rng)`
  - Vectorized particle motion update using a unicycle model with Gaussian control noise.
- `wrap_to_pi(angle)`
  - Wraps angle(s) to `[-pi, pi]`.

#### `raycast_sensor.py`

- `raycast_distances(poses, beam_angles, world, max_range)`
  - Casts beams to square/rectangular room walls and returns ray distances.
- `ray_sensor_model(particles, measured_ranges, beam_angles, world, sensor_noise_std, max_range)`
  - Computes vectorized log-likelihoods for raycast measurements.

#### `resample_particles.py`

- `resample_particles(particles, weights, rng)`
  - Systematic resampling; returns resampled particles and uniform weights.

#### `state_utils.py`

- `propagate_state(state, control, dt)`
  - Deterministic ground-truth propagation using the same unicycle kinematics.
- `estimate_state(particles, weights)`
  - Weighted mean pose estimate (circular mean for heading).

#### `compute_rmse.py`

- `compute_rmse(truth_xy, estimate_xy)`
  - Computes position RMSE for one or more 2D samples.

### Notes

- The range-only sensor model does not directly observe heading, so the baseline uses local particle initialization around the known start pose.
- The implementation is raycast-only now (square-room wall distances), which is better for MCL pose confirmation than the old landmark-range baseline.
- To reduce stalling/spin behavior, control dynamics use bounded jerk with a positive default linear speed floor and mean reversion toward nominal motion.
- The implementation is optimized for clarity + vectorized speed (NumPy), not maximum algorithmic complexity.
